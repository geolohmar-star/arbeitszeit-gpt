Workflow-Editor – Bedienungsanleitung
Toolbar – Uebersicht der Modi
Schritt-Modus

Klick auf die freie Flaeche erstellt einen neuen Knoten. Im Konfigurationspanel rechts dann Titel, Typ und Zustaendigkeit festlegen.

Verbinden-Modus

Ersten Knoten anklicken (Startpunkt), dann zweiten Knoten anklicken (Zielpunkt). Es wird eine Verbindung (Transition) erstellt. Danach die Transition anklicken um Bedingungen zu setzen.

Schere-Modus

Knoten oder Verbindung anklicken um sie zu loeschen.

Bearbeiten-Modus

Knoten oder Verbindung anklicken um Konfiguration zu oeffnen und zu aendern.

Knotentypen
Typ Einsatz Zustaendigkeit
task Manueller Schritt – jemand muss etwas tun (pruefen, genehmigen, bearbeiten) Direkte Fuehrungskraft, Abteilungsleitung, Team-Queue, Antragsteller, ...
auto Automatischer Schritt – wird sofort ohne Benutzeraktion ausgefuehrt Keine
decision Verzweigungspunkt – wertet Bedingungen aus und leitet in verschiedene Aeste Keine (automatisch)
sync Zusammenfuehrung – wartet bis mehrere parallele Aeste abgeschlossen sind Keine (automatisch)
Verzweigung (Wenn–Dann–Sonst)

Eine Verzweigung besteht aus:

  1. Einem Knoten vom Typ decision der ausgewertet wird
  2. Mehreren ausgehenden Transitionen mit verschiedenen Bedingungen
  3. Je einer Zielknoten pro Ast
Start
  ↓
[TL prueft] – task
  ↓ (genehmigt)
[Abzweig] – decision
  ├─→ Transition A: art == "fortbildung"  → [GF genehmigt] – task
  └─→ Transition B: art != "fortbildung"  → [Team ZG] – task
                                  ↓
                                  Ende
Bedingungstypen fuer Transitionen

Transition greift immer. Einsatz bei linearen Pfaden ohne Verzweigung.

Greift wenn der vorherige Task eine bestimmte Entscheidung hatte.

genehmigtBenutzer hat genehmigt
abgelehntBenutzer hat abgelehnt
weitergeleitetWeiterleitung an andere Person
rueckfrageRueckfrage gestellt

Prueft ein Feld des Antrags-Objekts gegen einen Wert. Drei Felder muessen ausgefuellt sein:

FeldBeschreibungBeispiel
Bedingungsfeld Feldname des Modells (Python-Attribut) art
Operator Vergleich: ==, !=, >, <, >=, <=, in ==
Bedingungswert Vergleichswert (interner DB-Wert, nicht Anzeigename!) fortbildung
Wichtig fuer Zeitgutschrift – Ganztaegige Fortbildung:
Der Anzeigename lautet "Ganztaegige Fortbildung bei individueller Arbeitszeit" – aber der interne Feldwert im Datenbankfeld art ist fortbildung.
Immer den internen Wert verwenden, nicht den Anzeigenamen!
Praxisbeispiele fuer Zeitgutschrift:
BedingungsfeldOperatorBedingungswertBedeutung
art == fortbildung Nur Fortbildungs-Antraege
art != fortbildung Alle ausser Fortbildung
art == haertefall Nur Haertefall-Antraege
art == ehrenamt Nur Ehrenamt-Antraege
fortbildung_aktiv == True Individuelle Vereinbarung bestaetigt

Fuer komplexe Bedingungen. Der Code muss True oder False zurueckgeben.

Verfuegbare Variablen: task, instance, content_object

Beispiel: 
content_object.art == "fortbildung" and content_object.fortbildung_aktiv
Schritt-fuer-Schritt: Abzweig anlegen
  1. Modus Schritt waehlen → Klick auf Flaeche → Knoten erstellen mit Typ decision, Titel z.B. "Art pruefen"
  2. Modus Verbinden → Von "TL prueft" zu "Art pruefen" verbinden → Bedingungstyp: Basierend auf Task-Entscheidung → genehmigt
  3. Modus Schritt → Zielknoten fuer Ast A anlegen (z.B. "GF genehmigt")
  4. Modus Schritt → Zielknoten fuer Ast B anlegen (z.B. "Team ZG")
  5. Modus Verbinden → Von "Art pruefen" zu "GF genehmigt" → Bedingungstyp: Feld-Wert → art / == / fortbildung
  6. Modus Verbinden → Von "Art pruefen" zu "Team ZG" → Bedingungstyp: Feld-Wert → art / != / fortbildung
  7. Workflow speichern → Graph-Modus muss aktiviert sein!
Hinweis Graph-Modus: Verzweigungen funktionieren nur wenn das Workflow-Template auf "Graph-basiert" gestellt ist. Im Speichern-Dialog die Option aktivieren.
