Facility-Workflows einrichten

Schritt-fuer-Schritt-Anleitung: Neue Stoermeldungen automatisch in die richtige Team-Queue weiterleiten.

Zum Workflow-Editor
Wie funktioniert das?

Wenn ein Mitarbeiter eine Stoermeldung erfasst, loest das System automatisch einen Trigger-Event aus – abhängig von der gewaehlten Kategorie. Existiert ein passendes WorkflowTemplate fuer diesen Trigger, wird die Meldung sofort als Task im Arbeitsstapel des zustaendigen Teams angelegt.

Mitarbeiter waehlt Kategorie "Elektro" und sendet Stoermeldung ab
  ↓
System: trigger_event = "stoermeldung_elektro_erstellt"
  ↓
WorkflowTemplate "Stoermeldung Elektro" wird gefunden (aktiv)
  ↓
WorkflowTask wird erstellt → TeamQueue "Facility – Elektro"
  ↓
Elektro-Teammitglieder sehen den Task im Arbeitsstapel und koennen ihn claimen und bearbeiten
Hinweis: Existiert kein WorkflowTemplate fuer eine Kategorie, passiert nichts Schlimmes – die Stoermeldung erscheint trotzdem in der Facility-Queue unter /facility/queue/. Der Workflow-Trigger ist ein zusaetzlicher Benachrichtigungskanal, kein Ersatz fuer die Facility-Queue.
Schritt 1 – Voraussetzung: TeamQueue-Eintraege anlegen

Der Workflow-Editor weist Schritte einer TeamQueue zu. Fuer jede Facility-Kategorie muss deshalb zuerst ein TeamQueue-Eintrag im Admin angelegt werden.

Admin: TeamQueue anlegen

Empfohlene TeamQueue-Eintraege:
Name Kuerzel Beschreibung Mitglieder
Facility – Elektro fac-elektro Elektrische Stoerungen, Beleuchtung, Steckdosen Elektro-Mitarbeiter eintragen
Facility – Sanitaer / Heizung fac-sanitaer Wasseranlagen, Heizung, WC Sanitaer-Mitarbeiter eintragen
Facility – Schlosser fac-schlosser Schloesser, Tueren, Schliesssystem Schlosser-Mitarbeiter eintragen
Facility – Schreiner fac-schreiner Moebel, Holzarbeiten, Einbauten Schreiner-Mitarbeiter eintragen
Facility – Maler fac-maler Malerarbeiten, Anstriche, Tapeten Maler-Mitarbeiter eintragen
Wichtig: Das Feld Antragstypen kann leer bleiben – es wird fuer Stoermeldungen nicht verwendet. Nur Name, Kuerzel und Mitglieder sind relevant.
Schritt 2 – WorkflowTemplate im Editor anlegen

Diesen Schritt 5x wiederholen – einmal pro Kategorie. Alle 5 Templates sind identisch aufgebaut, unterscheiden sich nur in Trigger-Event und TeamQueue.

1
Workflow-Editor oeffnen und neu starten

Zum Workflow-Editor → falls ein bestehendes Template geladen ist: "Leeren" klicken.

2
Schritt-Modus aktivieren und Knoten anlegen

Oben auf Schritt klicken. Dann auf die freie Flaeche klicken – ein Konfigurationspanel erscheint rechts. Dort eintragen:

Feld Wert
Schritt-Titel Stoermeldung bearbeiten
Typ task
Zustaendige Rolle Team-Queue
Team auswaehlen z.B. Facility – Elektro (aus dem Dropdown)
Frist (Tage) 3 (oder je nach Anforderung)

"Schritt hinzufuegen" klicken. Der Knoten erscheint auf der Flaeche.

3
Speichern-Dialog oeffnen

Oben auf Speichern klicken. Im Dialog folgendes eintragen:

Feld Wert
Template-Name z.B. Stoermeldung – Elektro
Trigger-Event Dropdown → Gruppe Facility Management → passenden Eintrag waehlen (Tabelle unten)
Aktiv Haken setzen

"Speichern" klicken. Fertig fuer diese Kategorie.

Referenz: Trigger-Events und zugehoerige TeamQueues
Kategorie Trigger-Event (im Dropdown) TeamQueue (Kuerzel)
Elektro stoermeldung_elektro_erstellt fac-elektro
Sanitaer / Heizung stoermeldung_sanitaer_heizung_erstellt fac-sanitaer
Schlosser stoermeldung_schlosser_erstellt fac-schlosser
Schreiner stoermeldung_schreiner_erstellt fac-schreiner
Maler stoermeldung_maler_erstellt fac-maler
Schritt 3 – Setup testen
  1. Als normaler Mitarbeiter anmelden → Neue Stoermeldung erfassen → Kategorie "Elektro" waehlen → Pflichtfelder ausfullen → Absenden.
  2. Zur Erfolgsmeldung-Seite: Betreff notieren (z.B. STR-00001).
  3. Als Elektro-Teammitglied anmelden → Arbeitsstapel aufrufen → Der Task sollte sichtbar sein mit Link zur Stoermeldung.
  4. Task aus dem Team-Stapel claimen → Facility-Queue → Stoermeldung erledigen → Kommentar eingeben.
  5. Als Melder anmelden → Meine Stoermeldungen → Status sollte "Erledigt" zeigen.
Haeufige Probleme
Problem Ursache Loesung
Task erscheint nicht im Arbeitsstapel WorkflowTemplate existiert nicht oder ist inaktiv, oder Trigger-Event stimmt nicht exakt ueberein Im Editor: Template laden → Trigger-Event pruefen. Exakt: stoermeldung_elektro_erstellt (Gross-/Kleinschreibung beachten)
Task im Arbeitsstapel, aber kein TeamQueue-Eintrag sichtbar WorkflowStep hat keinen zustaendig_team-Wert Im Editor: Knoten im Bearbeiten-Modus oeffnen → "Team-Queue" eintragen
Stoermeldung erledigt, aber WorkflowTask bleibt offen Seltener Fehler in _schliesse_facility_workflow_task() Task manuell im Arbeitsstapel als erledigt markieren. Logs pruefen.
TeamQueue erscheint nicht im Dropdown des Editors TeamQueue-Eintrag fehlt im Admin Admin → TeamQueue → Eintrag anlegen, dann Seite im Editor neu laden
